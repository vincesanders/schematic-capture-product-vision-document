Problem(s) Addressed

  • Front-End Bug Problems
    • landing page loads slowly - image too large?
    • landing page - footer covers login form on short screen
    • Components component in front end references component.id when it should reference the component id
    • Schematic cannot be seen on Web portion of the app

  • Front-End Feature Problems
    • tab icon in browser displays React logo
    • Users must be able to organize their photos and notes for their job sheets and projects
    • Nothing shows up to show user a schematic has been selected to be sent to cloud
    • Only see 4 columns of the csv file when parsed to json on the front end... possible scroll bar issue / too many columns to view all at once

  • Back-End Feature Problems
    • Back-end documentation is not explicit as to the cause of its errors
    • Users seem to have no ability / rights to access a client that isn't assigned to them, maybe admins do?
      • Users can create new clients, but when attempting to create a new jobsheet they are given a 404... figure out why (backend?) says unassigned, but Bob (test user) can access test project 1 without issues, seems to have access on mobile as well, probably not same test project 1 though

  • Full-Stack Problems
    • Hardly any testing has been done on the front-end or back-end
    • Code is hard to navigate

Solution

  • Front End Bug Fixes
    • Change CSS on landing page to ensure proper layout for shorter screens.
    • Resize image on landing page
    • add all other columns of the csv to the front end display, work from there to decide what to do if the table is too big to work on from a desktop. Click and zoom in?
    • Change column referenced in component to display the proper data
    • ability to see the schematic in the web version

  • Back End Bug Fixes
    • figure out what is happening with the error notification system that is in place right now, and how it will be possible to display errors multiple times even if they are the same error

  • Front End Feature Revisions
    • Change icon on browser tab
    • Give users the ability to upload new pictures or saved pictures to the job sheet or project
    • add a textbox showing the name of the schematic that you are uploading from your computer

  • Back End Feature Revisions
    • Update documentation to be explicit about reason for each error.
    • User is checked if their organization is matched with their user_organization when trying to access a client and that clients jobsheet. When creating a jobsheet this middleware is in place. Basically, there is no way for a new client to be made, and for that specific user (or admin) to add a jobsheet. You get a 403 in projects when accessing them. Remove this middleware, and the user_organization table. We will eventually need a user_organization table when implementing a new organization in the app, not necessary for this release canvas
      • Users may need to have access to specified clients, confirm with Wes that this is not a feature that he wants. In the case that he does, must add a join table that references the client, and the user. Place middleware checking to if there is relation between user and client

  • Full-Stack Revisions
    • Implement testing
    • Refactor code to be more readable

Objectives Supported

  • Reduce time spent capturing machine component pictures offline
  • Increase user task completion rate by 100%
  • Decrease on-boarding time for new employees by 50%

Key Results

  • Reduce load time around site by 25%
  • Users are able to complete their tasks 2 times faster than previously adopted method

End Results

  • Tracking around the application has become a breeze with little downtime, at least 25% increase in fluidity, as the previous iteration was inoperable in comparison
    • Completed by cleaning up messy file structure - removing what was not necessary, making routes as simple as possible, and removing the JUNK!
  • Users can see a huge increase in productivity from their previous work flow as shown by Wes
    • this was the purpose of our application - the reason Wes wanted this to be done. We as a team feel like we have been able to adopt his vision as closely as possible to almost have delivered him a usable product for him and his team to begin to use so they can start cutting time spent at a desk thanks to Schematic Capture.
