Problem(s) Addressed

  • Users must be able to organize their photos and notes for their job sheets and projects
  • Nothing shows up to show user a schematic has been selected to be sent to cloud
  • Schematic cannot be seen on Web portion of the app
  • User stays logged in even when token is expired
  • Cannot generate new token without using postman ATM
  • If getting multiple errors when trying to login (invalid credentials) you will only get a notification for a 403 the first time, all other times you will get back no error
  • Back-end documentation is not explicit as to the cause of its errors
  • Clients are indistinguishable from each other besides their name... what if you have 1000+ clients
    • Search or filter client list
  • Only see 4 columns of the csv file when parsed to json on the front end... possible scroll bar issue / too many columns to view all at once
  • Users and admins have the same permissions ATM...
  • Cannot create new clients (as admin)
  • Cannot sign in with google account
  • Cannot 'delete' inactive clients (as admin)
  • landing page loads slowly - image to large?
  • landing page - more info about product / app
  • long wait on initial Google sign in - why?
  • tab icon in browser displays React logo
  • landing page - footer covers login form on short screen
  • Front-end: Components component in front end references component.id when it should reference the component id
  • Front-end: Technician has access to / can see options / buttons for functionality he doesn't have access to.
  • Back-end: The token cannot be refreshed.
  • Developers do not have access to a super-user / admin account for testing
  • Hardly any testing has been done on the front-end or back-end
  • Build functionality to edit photos taken by technician to only be allowed before they're uploaded to cloud
  • No functionality for accepting an image to PUT / POST in jobsheet data, will eventually be sent to dropbox
  • Rename tables, user becomes technician
  • add technician_id to jobsheet that references assigned technician's id

Solution

  • Give users the ability to upload new pictures or saved pictures to the job sheet or project
  • GET schematic from firebase database (or if changed to dropbox, same to dropbox)
  • add a textbox showing the name of the schematic that you are uploading from your computer
  • if your token is expired, redirect to homepage - also make length of valid token way longer than it is currently (1hr current)
  • Invite user is a difficult process so far, the token is too long for the email, the invitation link that allows for you to create a new account doesn't work, might need a whole overhaul
  • figure out what is happening with the notification system that is in place right now, and how it will be possible to display errors multiple times even if they are the same error
  • search feature on the clients, add profile picture maybe too
  • add all other columns of the csv to the front end display, work from there to decide what to do if the table is too big to work on from a desktop. Click and zoom in?
  • Resize image on landing page
  • Work closely with UX designer to develop more content for the landing page
  • Optimize google sign in or worker threads (multiple threads to handle async operations)
  • Change icon on browser tab
  • Change CSS on landing page to ensure proper layout for shorter screens.
  • Add functionality to delete token or refresh it after it expires
  • Change column referenced in Component component to display the proper data
  • Add functionality to change visibility of content not accessible by the user
  • Add checkRole middleware to the back-end
  • Make the expiration time of a token long enough to cover an entire working day
  • Create a super-user / admin account for testing
  • Update documentation to be explicit about reason for each error.
  • Implement testing on front and back-end
  • Backend - create the ability to push to dropbox though the backend, store url here as well

Objectives Supported

  • Giving users an easy and intuitive product to help organize their work
  • Gives users the ability to complete desk work on their phone to allow for less extra time off site
  • gives users a tool that makes their job easier to work through a flow
  • Allows new employees to better understand the job at hand, has ability to see all the necessary information on their phone

Key Results

  • Refactor when necessary to make functionality work
  • Bug fixes
  • Users and Admins have specified abilities that are distinguished based on their role
  • Can go through basic flow with limited bugs
  • Reach the testing 40% goal
  • Reach the B rated maintainability goal
  • 90+% of users feel that the product is easy to use and intuitive
