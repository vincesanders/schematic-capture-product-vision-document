Copy of Vision - Problem & Objectives

Problem Core

Because there is no effective way to document maintenance of machines many times issues are overlooked and documentation is disorganized leaving the machines vulnerable to damage and increased down time for repairs.

Problem Side Effects

  1. Time wasted trying to determine where a component is what without visual documentation.
  1. Money wasted when the maintenance guys replace the wrong component.
  1. Both of the above cause an increase in downtime which in many industries is unrecoverable.
  1. In addition, if there are multiple instances of the same component, there may be variations (manufacturers, etc.) that need to be noted and illustrated.
  1. Outdated information leads to unproductive work environment.
  1. Increased onboarding time spent with new employees trying to get familiarized with machinery.

Objectives

  1. Reduce time spent capturing machine component pictures offline
  1. Create the ability to enter or change data on the job sheet when capturing data to improve the accuracy and integrity of the schematic and components.
  1. 3. Build or improve functionality that allow technicians to know what and where each component is located and update the schematic if necessary.
  1. Develop an application that is well integrated end to end.
